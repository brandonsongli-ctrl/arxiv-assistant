# AGENTS.md

## Project Guidelines

1. Every update (code, docs, config) must be pushed to GitHub after local verification.
2. Default post-update flow:
   - Run relevant checks/tests.
   - Commit with a clear message.
   - Push to `origin/main`.
3. Do not leave update commits only in local repository unless explicitly requested.
4. If push fails, report the exact error and stop until it is resolved.

## 项目准则（中文）

1. 每次更新（代码、文档、配置）在本地验证后都必须推送到 GitHub。
2. 默认更新后流程：
   - 运行相关检查/测试；
   - 使用清晰提交信息进行 commit；
   - 推送到 `origin/main`。
3. 除非有明确要求，否则不允许只在本地保留更新提交。
4. 如果 push 失败，需报告具体错误并暂停后续提交流程。
